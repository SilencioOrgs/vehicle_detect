# Vehicle Detection & Counter — Python Dependencies
# ==================================================
# Works on both Windows/Desktop and Raspberry Pi 4B (Bookworm OS)
#
# ---- AGENTIC SETUP PROMPT (for AI Copilot on Raspberry Pi) ----
# When on a Raspberry Pi, run these commands in order to set up
# the project. Execute each step and verify before moving to next:
#
#   Step 1: Install system dependencies
#     sudo apt update && sudo apt install -y python3-pip python3-venv \
#       libopencv-dev python3-opencv libatlas-base-dev libhdf5-dev \
#       libharfbuzz-dev liblapack-dev gfortran
#
#   Step 2: Create virtual environment with system site-packages
#     python3 -m venv --system-site-packages .venv
#     source .venv/bin/activate
#
#   Step 3: Upgrade pip and install Raspberry Pi support packages
#     python -m pip install --upgrade pip setuptools wheel
#     pip install -r requirements-rpi.txt
#
#   Step 4: Install CPU-only PyTorch and Ultralytics
#     pip install torch torchvision --index-url https://download.pytorch.org/whl/cpu
#     pip install --no-deps ultralytics
#     python -c "import torch; print(torch.__version__)"
#
#   Step 5: Verify everything works
#     python -c "from ultralytics import YOLO; print('YOLO OK')"
#     python -c "import cv2; print('OpenCV', cv2.__version__)"
#
#   Step 6: Run the detection
#     python main.py
#
#   Notes:
#     - Raspberry Pi uses the apt-provided OpenCV build (`python3-opencv`)
#       through the virtualenv's `--system-site-packages` setting.
#     - Install torch before ultralytics on Raspberry Pi so pip does not try
#       to resolve GPU/CUDA torch dependencies for a CPU-only board.
#     - `requirements-rpi.txt` carries the non-torch Ultralytics runtime deps,
#       so `ultralytics` itself is installed with `--no-deps`.
#
# ---- Desktop install (simple) ----
#   python -m pip install -r requirements.txt
# ==================================================
